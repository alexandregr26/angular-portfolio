Project Description

My programming foundation and computer knowledge were formed by creating and implementing software applications in low-level programming languages such as Assembly, C, and C++. This was thanks, in no small part, to the challenges of the third-year course ECE344: Operating Systems at the University of Toronto. This course presented projects that made us dig deep into threads, processes, and concurrency. In one major assignment, I built a preemptive, user-level thread package, incorporating mutex locks and condition variables to simulate concurrency on processors.

In another unrelated but subsequent lab, I coded a caching, multi-threaded web server, enabling parallelism. The approach involved creating a pool of worker threads that managed incoming HTTP requests and implementing a memory-based file caching system to store recently requested files. Completing these projects not only sharpened my technical skills but also gave me a fascinating peek into the inner workings of computers, leaving me excited to dive even deeper into this topic in the near future.

The source code for the thread package and the caching web server are available on my GitHub page but on a private repository to support academic integrity. Please contact me if you would like to know more.
