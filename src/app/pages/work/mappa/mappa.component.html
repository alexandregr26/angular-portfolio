Project Description

As part of my second-year engineering design course at the University of Toronto, ECE297, I was part of a team that took on a project to build a Geographic Information System (GIS) application. Working exclusively with C++, we aimed to create an application similar to Google Maps. We did this by leveraging APIs to query a geographic information database and employing a graphics library to display such information to the users, thus bringing the GIS application to life. Throughout the project, my biggest priority was designing a user-friendly interface and creating a seamless user experience, reflecting my belief in the significance of aesthetics and functionality in the engineering design process.

The Team

The first stage of the project was team formation. In this stage, all team members gathered to align on project goals, draft a team charter, and assign individual roles. We prioritized open communication and quickly established weekly meetings to ensure everyone was on the same page and to address any challenges as soon as possible. Regarding code management, we all used Git - for the first time, preventing conflicts in the project's source code. This focus on communication and collaboration laid a strong foundation for our journey throughout the GIS application development.

Development

With the support of a Teaching Assistant (TA), who provided guidance and reviewed our work at every milestone, we ensured our technical decisions aligned with project goals. This iterative feedback loop enhanced our communication with technical supervisors. Simultaneously, under the guidance of a Communication Instructor, we learned to present our progress and final project to less technical clients. This instructor bridged the gap, helping us convey complex technical aspects in an accessible way. This dual guidance prepared us for real-world scenarios, emphasizing the importance of effective communication in software engineering projects.

Powerful Search Bar

We built a search bar with the following capabilities:

  • Allow users to find their destination using partial street names or misspelled names
  • Locate and highlight the intersection of two given streets
  • Find the shortest driving or walking path between two locations
  • Delete any past search-related data to improve the map's responsiveness

100% Customizable

We wanted to give complete control to the user by making every feature on the map customizable. One of our ideas as a result of this was a mode that allows the user to fully customize the map colour scheme and filter specific content from the map.

Visual Features

  • Street names displayed at proper zoom levels
  • Major roads are differentiated from minor roads
  • Colour-coded points of interest
  • Visible street directions
